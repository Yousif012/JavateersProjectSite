User Stories

1: Creating Equations

As a student, I want to be able to write
complex equations into my notes.


Elaboration: Users should be able to quickly and easily create
a wide range of equations and put them in their notes. It should be
easy to do to avoid confusion and quick enough that students can keep up
while writing notes during a lecture.


Constraints: none


Effort Estimation: 8 person-hours


Acceptance Test: Create five different equations of increasing complexity
using a built-in tool and ensure they all render correctly.

2: Organize notes


As a user of the note-taking app, I want to be able to organize my notes
by subject and date so that I can easily find and review my notes.


Elaboration: The app should allow the user to create and edit subject categories
and assign notes to those categories. The user should also be able to sort notes by date,
both chronologically and reverse-chronologically. The app should provide a visual indication
of which notes have been categorized and which notes are unsorted. The user should also be
able to filter notes by subject and date range.


Constraints: The app should be intuitive and easy to use, with a minimal learning curve.
The app should not allow the user to create duplicate categories or assign notes to
multiple categories.


Effort Estimation: 20 person-hours


Acceptance Test: Create and edit subject categories and assign notes to those categories.
Sort notes by date in both chronological and reverse-chronological order. Check that the
app visually indicates which notes have been categorized and which notes are unsorted.
Filter notes by subject and date range. Ensure that the app does not allow the user to
create duplicate categories or assign notes to multiple categories.

2.1: Improved Sorting Options

As a user of the note-taking app, I want to be able to sort my notes by additional criteria such as title, priority, and type, so that I can easily find and organize my notes according to my needs.


Elaboration:

  • The app should allow the user to sort notes by title, priority, and type in addition to subject and date.
  • The user should be able to choose which criteria to use for sorting and the order in which to sort them.
  • The app should provide a visual indication of the sorting criteria being used and the order of sorting.


Constraints: The app should maintain the same level of intuitive and easy-to-use interface and shouldn't affect the app's performance.


Effort Estimation: 10 person-hours


Acceptance Test: Verify that the app allows sorting notes by title, priority, and type. Verify that the user can choose the sorting criteria and order. Verify that the app provides a visual indication of the sorting criteria being used and the order of sorting.

2.2: Note Sharing

As a user of the note-taking app, I want to be able to share my notes with others, so that I can collaborate and work on notes with other people.


Elaboration:

  • The app should allow the user to share individual notes or entire categories of notes with others via email or other messaging apps.
  • The user should be able to set permissions for each shared note or category, such as read-only or edit access.
  • The app should provide a notification to the user when a shared note or category is edited by someone else.


Constraints: The app should maintain user privacy and security, ensuring that only the intended recipient has access to the shared notes.


Effort Estimation: 15 person-hours


Acceptance Test: Verify that the app allows the user to share individual notes or entire categories of notes.
Verify that the user can set permissions for each shared note or category.
Verify that the app provides a notification to the user when a shared note or category is edited by someone else.
Verify that the app maintains user privacy and security.

2.3: Note Formatting

As a user of the note-taking app, I want to be able to format my notes and change metrics such as the font size or the color.


Elaboration:

  • The user should be able to change the font color and choose among many RGB colors.
  • The user should be able to change the font size and be able to make the text smaller or bigger.
  • The user should be able to make the text both in italics and bold.
  • The user should be able to align the text to left, center, or right.


Constraints: We should set limits to font size to not over-enlarge the text so that it becomes bigger than the screen.


Effort Estimation: 10 person-hours


Acceptance Test: Verify that the app allows the user format the text.
Verify that the user cannot set the font size above a certain value.

3: Adding Charts

As a financial analyst, I want to keep track of trends by keeping charts in my notes.

3.1: Adding charts to one's notes

As a financial analyst, I want to easily take CSV files and convert them into charts in my own notes.

Elaboration: Users should be able to import CSV file types to their notes and they should be able to choose which columns they want to plot and what data they should include.

Constraints: Limit the amount of data that can be imported.

Effort Estimation: 5 person-hours

Acceptance Test: Import CSV files into notes and make the first two columns a Line Chart.

3.2: Allowing user to mark points on the chart

As a financial analyst, I want to mark certain important points on my chart so that I can improve my presentation.

Elaboration: Users should be able to choose a point by using x-y coordinates and marking the point with an X.

Constraints: Only allow markers on the graphs themselves.

Effort Estimation: 7 person-hours

Acceptance Test: Being able to mark any point on a chart.
