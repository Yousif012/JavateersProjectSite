Architecture

Project Overview

Our team is developing a note-taking app for android geared toward students. In a fully realized version of our app, users will be able to take notes anywhere on the page, draw freely, make graphs and charts, plot functions, write complex math equations, and make tables of values all very quickly and easily.

Architecture Considerations

1. Client-Server

  • Client will be the user opening our application from a mobile phone.
  • Client will interact with a data storage server where they get access to their saved notes in the cloud.
  • Client will have special access key for their own notes.

2. Repository

  • Central data storage being the database where we store the user's notes.
  • Components being the users who access the Central data storage to retrieve their own note.

Architecture: Repository

Conclusion

Chosen architectural style: Repository.
