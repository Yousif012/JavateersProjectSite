Test Report

During the development of the Note Taking App, we performed several unit and component tests to ensure the proper functioning of individual parts of the system. We used JUnit, an open-source unit testing framework, for the testing of the Java components of the application. We tested the following components separately:

  • Text entry feature: We tested the text entry feature by creating various text input scenarios and verifying that the text is correctly saved and displayed.
  • Drawing feature: We tested the drawing feature by creating different drawing scenarios and verifying that the drawings are correctly saved and displayed.
  • Latex feature: We tested the Latex feature by creating different equations and verifying that they are correctly rendered and displayed.
  • Graphing feature: We tested the graphing feature by creating different graphs and verifying that they are correctly rendered and displayed.
  • Auto-save feature: We tested the auto-save feature by verifying that the notes are automatically saved at regular intervals.
  • Local and Cloud storage: We tested the storage features by verifying that the notes are correctly saved and retrieved from both local and cloud storage.

We selected the test data and scenarios based on the expected requirements of the application. We also used edge cases to ensure that the application could handle unexpected scenarios. We used JUnit's assert methods to check the expected output against the actual output.

2. System Testing

We performed system testing by testing the entire Note Taking App as a whole. We selected a range of test cases and test data to ensure that the app functioned correctly in various scenarios. We also used Monkey Testing, a random testing technique that generates random inputs, to test the app's robustness.

We used Android Studio's built-in tools for testing, including the Android Emulator and the Android Debug Bridge (ADB). We also used Firebase Test Lab, a cloud-based testing infrastructure, to test the app on multiple devices with different configurations. We used TestFairy, a mobile testing platform, to capture user feedback and track issues.

During system testing, we verified the proper functioning of all the features, including text entry, drawing, Latex, graphing, auto-save, and storage. We also tested the app's performance and stability under different conditions.

3. Acceptance Testing

We used the acceptance criteria on our user stories to drive our system testing. The following user stories were tested for acceptance:

  • As a user, I want to be able to create text notes, so that I can quickly jot down my ideas.
  • As a user, I want to be able to draw sketches, so that I can visually express my ideas.
  • As a user, I want to be able to write mathematical equations in Latex, so that I can use the app for scientific purposes.
  • As a user, I want to be able to create graphs, so that I can visualize my data.
  • As a user, I want the app to automatically save my notes, so that I don't lose my work.
  • As a user, I want my notes to be stored locally and in the cloud, so that I can access them from anywhere.

All user stories were tested and accepted as completely implemented. We verified that all the acceptance criteria were met, and the features functioned as expected. Any issues or bugs discovered during acceptance testing were resolved before the final release of the app.

In conclusion, we performed rigorous testing at the component, system, and acceptance levels to ensure the proper functioning of the Note Taking App. We used various tools and techniques to test the app's functionality, performance, and robustness, and we accepted all the user stories as completely implemented.
